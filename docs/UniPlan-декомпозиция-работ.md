## Эпик 2. Календарь и date patterns

## Эпик 3. Layer 2 — расписание внутри блока/недели (coursett)
- 🔄 3.1 **Боевой адаптер домен → coursett** (Section/Location/Instructor/DatePattern → Lecture/Room/Instructor-Constraint/TimeLocation).
- ⬜ 3.2 Обратный маппинг результата → `Assignment` в БД (под `ScheduleVersion`).
- ⬜ 3.3 Конфигурация решателя (критерии, веса, таймауты) под наши требования.
- ⬜ 3.4 Асинхронный прогон (джоба) + статус/прогресс.
- ⬜ 3.5 Параллельный прогон по блокам.
- 🔮 3.6 Студенческие/групповые конфликты (группа как «студент»), distance-конфликты.

## Эпик 4. Layer 1 — цикловая ротация (Режим A: валидация)
- ⬜ 4.1 Ввод ротации (`RotationAssignment`, `DisciplineAssignment`).
- ⬜ 4.2 Валидатор: ёмкость кафедр по блокам, отсутствие наложений, покрытие всех дисциплин, непрерывность диапазонов групп.

- 🔮 4.4 Режим B — авто-генерация ротации (кастомная IFS-модель Layer 1).
